## 3. 접근성 고려 작업
접근성 고려 작업을 진행하였다.  


WAI-ARIA를 사용하여  
웹 접근성을 보완하기 위한 속성 표준을 지키려 했다.

수업시간에 배웠던 WAI-ARIA가 잘 기억나지 않아..
AI와 구글링을 하여 어떤부분에 어떤 기법을 쓰면 좋을지 고려해 보았다.

1. aria-labelledby
    이 요소(영역)의 이름은 '다른 요소의 텍스트'입니다
    (이 요소의 제목(이름)을 다른 요소의 텍스트로 지정할 때 사용/주로 section, nav, dialog 같은 의미 있는 영역)

2. aria-hidden="true" 
    화면에 보이지만 스크린리더기에서는 무시
    (장식용 아이콘, 이모지, 반복되는 텍스트 등에 사용 / 의미 없는 정보를 스크린리더가 읽지 않게 함 /시각적 요소와 접근성 정보 분리)

3. aria-live="polite"
    텍스트가 바뀌면 스크린리더가 예의 있게 알려줌
    (JS로 내용이 동적으로 변경될 때 사용 / 사용자의 현재 읽기를 방해하지 않고, 읽고 있던 게 끝나면 변경 내용을 읽어줌 / 알림, 상태 메시지 등에 적합)
