# 과제 수행 과정 기록

## 1. 과제 초기 세팅
스크립트 작업 이후 마크업을 진행할 예정


## 2. 스크립트 작업 전 구현 방법 고민
버튼 클릭 시 모든 버튼과 콘텐츠의 active 클래스를 제거한 뒤,  
클릭된 버튼에 active 클래스를 추가하고,  
해당 버튼과 매칭되는 콘텐츠에 active 클래스를 적용하는 방식으로 구현하고자 생각


이를 위해 addEventListener를 사용한 클릭 이벤트 처리와  
classList를 활용한 클래스 추가,제거 방식으로 진행할 계획


## 3. 스크립트 작업
addEventListener로 클릭 이벤트를 적용했으나 오류가 발생했고,  
콘솔 로그도 출력되지 않았음.


콘솔 메시지가 찍히지 않는 것으로 보아  
이벤트가 연결되기 이전 단계부터 정상 동작하지 않는다고 판단함.


확인 결과, 원인은 querySelectorAll() 사용에 있다는걸 깨달음.  


querySelectorAll()은 여러 요소를 NodeList 형태로 반환하므로,  
이 묶음 자체에 addEventListener를 직접 적용할 수 없음을 파악함.


여러 버튼 중 어떤 버튼이 클릭될지 모르기 때문에  
전체 버튼을 가져온 뒤 각각에 이벤트를 연결해야 했고,  
이를 위해 반복문을 적용하기로 생각함.


for...of 문으로 NodeList를 각 요소에 클릭 이벤트를 등록했고,  
이벤트 내부에서 classList로 active 클래스 추가/제거를 제어하도록 구현.


이를통해 아래 내용을 다시 리마인드함  
 - querySelector는 1개의 요소를 선택할때 사용  
 - querySelectorAll은 여러개의 요소를 선택할때 사용  
 - addEventListener는 DOM 요소(Element) 1개에만 사용 가능


그 다음으로 내가 원하는건!  
클릭한 버튼과 매칭되는 컨텐트를 어떻게 노출할 수 있을까를 고민...


this 키워드를 사용하여..  
클릭한 이벤트를 체크 해야하나 생각함.


this를 사용하는 방법에 대하여 개념 공부뒤 진행하려고 할까?!  
배운공부 내용에서 사용할수있는 방법이 있는가?!를 고민함


노트정리한걸 훑어 보던 중 forEach를 사용하면서 index를 체크할 수 있다고 생각남  
기존 for...of를 forEach사용으로 변경하고,  
컨텐트의 index를 체크하여 클래스를 붙이는 방식으로 코드를 수정함.
