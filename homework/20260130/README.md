# 과제 수행 과정 기록

## 1. 과제 초기 세팅
구조를 분석한 뒤 과제를 수행 할 예정.  


JS로직 구현이 끝난 후  
시간적 여유에 따라 마크업을 진행하겠습니다.


## 2. 스크립트 작업 전 구현 방법 고민
1. 플랜카드(요소) 클릭시, 클릭한 플랜카드를 찾아서 selected 클래스를 붙인다.
2. 그외 다른 플랜카드(plan-card를 가진 article태그)에 selected 클래스가 있다면, selected 클래스를 제거해준다.
3. selection-info 클래스를 가진 요소(div태그)를 찾아서 show 클래스를 붙여준다.
4. 내가 클릭한 요소(플랜카드)를 체크하여, 그 플랜카드의 이름자리(data-id="plan-price"값, data-id="plan-price"값을 가진 태그)에 텍스트를 넣어준다.


## 3. 스크립트 작업
지난 과제와 비슷한 기능으로  
다른 방식으로 진행할 방법이 생각나지 않아..


우선 지난 과제 방법과 동일하게 진행후!  
작성코드에 대해 차근차근 리펙토링을 해보자고 생각하였다.  
(근데.. 생각보다 지난주에 작업을 했음에도 불구하고..  
움직이지 않는 손가락 덕분에 당황스러웠다..)


작업의 진행은  
1. querySelectorAll로 가져와 forEach문을 작성후!  
forEach문 내부에 클릭이벤트리스너를 구현하고자 했다.


2. 클릭이벤트리스너 내부에 어떤 작업을 해야 하는지 생각이 나지 않아..  
지난번 작업해놓은 코드를 보니.. 두가지 작업을 진행을 해야한다는걸 파악 할 수있었다.  
첫번째, 내부에 forEach문을 재사용하여 클래스 제거  
두번째, index로 현재 클릭한 값을 체크해서 클래스 추가


3. 첫번째, index로 클릭한 값을 체크해서 사용한걸 보고!  
지난번 과제시 this 키워드와 고민 했었던걸 생각이 났다.  
마침 수업시간에! this키워드를 배웠으니! this를 사용하겠다고 결정했다.


4. this키워드를 자신있게 사용했다 > 에러 메세지 발생!!  
this를 변수에 담아, 콘솔 찍어보니! window가 뜸.  
전역에서의 this이구나를 파악했지만,  
전역인데.. 뭐?! 왜 그러는지?! 어떻게 해결방안이 생각 나지 않았다.


AI를 의지 해볼까 하다가! 노트정리 한 부분을 체크했다.  
노트정리를 보니~ this키워드는 화살표 함수 내부에서 사용하면! 전역에서 만들어진다!고 써 있었다.


this 사용시 처리방향에 대하여 노트정리를 살펴보며 생각한 해결방안은 2가지가 있었는데  
첫째, addEventListner의 리스너 영역을 함수로 만들어서 사용한다.  
둘째, 이벤트객체.cuttrtTarget을 사용하여! 이벤트가 발생한 대상을 체크한다.


리스너 영역을 함수로 만들어 사용하는게 재사용성에 있어서도 좋아~  
함수를 만들어 사용해볼까 하였는데.. 생각보다 오래걸릴듯하여  
두번째 방법인 currentTarget으로 진행하였다.


5. 클래스를 제거하는 방법은 어떻게 하면 좋을까 고민을 하다가  
변수를 만들어 selected클래스를 가지고 있는 태그를 담은후  
if문 사용하여, selected클래스를 가지고 있으면 클래스 제거할 수 있게 작업 하였다.


4. 템플릿 리터럴 내부, 선택한 플랜카드의 내용의 데어터를 불러오는 작업!  
여기서 forEach를 사용해 인덱스를 체크하여 넣어주는 방법만 생각이 나서! 인덱스 사용할 계획 이었다.


근데 생각보다 index로 데이터를 불러오는게 순조롭지 않았다.  
그래서 고민을 하다가~ html내부의 data-*가 있다는걸 확인하고!  
dataset을 사용하여 데이터를 불러왔다.
